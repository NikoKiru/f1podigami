# Security Policy

## Supported Versions

Security fixes are provided for the latest release.

## Reporting a Vulnerability

If you believe you have found a security issue, please do not open a public issue.

Report it privately by contacting the repository maintainer with:

- a clear description of the issue
- steps to reproduce
- any relevant logs, screenshots, or proof of concept
- the affected version or commit

Please allow time for review and remediation. Acknowledgements and fix timelines may vary based on severity and impact.

## Security Best Practices

- Keep dependencies up to date
- Review changes before merging
- Avoid committing secrets, tokens, or credentials
- Validate and sanitize any external input
